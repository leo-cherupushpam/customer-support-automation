# Customer Support Knowledge Base

## Password Reset Guide

### Overview
Help customers who have forgotten their password or need to change it for security reasons.

### Steps
1. Go to the login page and click "Forgot Password"
2. Enter your registered email address
3. Check your inbox for a reset link (may take up to 5 minutes)
4. Click the link and set a new password (minimum 8 characters, one uppercase, one number)
5. Log in with your new password

### Tips
- Check your spam/junk folder if the email doesn't arrive
- The reset link expires after 24 hours
- Contact support if you no longer have access to your registered email

## Refund Policy

### Overview
Our refund policy for subscriptions, one-time purchases, and add-ons.

### Subscription Refunds
- Monthly plans: refundable within 48 hours of charge if no usage occurred
- Annual plans: pro-rated refund within 14 days of purchase
- After these windows, refunds are issued as account credits

### One-Time Purchase Refunds
- Digital products: no refund once downloaded or accessed
- Physical products: return within 30 days for a full refund (shipping cost excluded)

### How to Request a Refund
1. Go to Account > Billing > Transaction History
2. Click the transaction you want refunded
3. Select "Request Refund" and complete the form
4. Allow 5-10 business days for credit card refunds

## Technical Troubleshooting

### Overview
Step-by-step guide for diagnosing and resolving common technical issues.

### General Troubleshooting Steps
1. Clear your browser cache and cookies
2. Try a different browser or incognito/private mode
3. Disable browser extensions temporarily
4. Check our status page at status.example.com for outages
5. Restart the application or refresh the page

### Common Errors
- **Error 403 Forbidden**: Your session may have expired — log out and back in
- **Error 500 Internal Server Error**: Likely a temporary server issue — wait 5 minutes and retry
- **Blank/white screen**: Hard refresh (Ctrl+Shift+R / Cmd+Shift+R) usually resolves this

### When to Contact Support
If the issue persists after these steps, contact support with:
- Your account email
- Browser and OS version
- Screenshot or screen recording of the error

## Performance Issues

### Overview
Resolving slow load times, timeouts, and degraded performance.

### Quick Checks
1. Test your internet connection speed (minimum 5 Mbps recommended)
2. Check status.example.com for any active incidents
3. Close unused tabs and applications to free up memory

### Browser Optimizations
- Update your browser to the latest version
- Disable hardware acceleration if you experience graphical glitches
- Limit browser extensions — some can significantly slow page rendering

### Network Issues
- Try a wired connection instead of Wi-Fi
- Disable VPN temporarily to rule out routing issues
- Contact your ISP if problems are consistent

### When to Report Performance Issues
If dashboards take more than 10 seconds to load consistently, submit a support ticket with:
- The URL of the slow page
- Time of occurrence and timezone
- Browser, OS, and network type

## API Documentation

### Overview
Getting started with our REST API for developers and integrators.

### Authentication
All API requests require a Bearer token:
```
Authorization: Bearer YOUR_API_KEY
```
Generate an API key at Account > Developer > API Keys.

### Base URL
```
https://api.example.com/v1
```

### Rate Limits
- Free tier: 100 requests/minute
- Pro tier: 1,000 requests/minute
- Enterprise: custom limits

### Common Endpoints
- `GET /tickets` — list tickets
- `POST /tickets` — create a ticket
- `GET /tickets/{id}` — get a specific ticket
- `PATCH /tickets/{id}` — update a ticket

### Error Codes
- 400 Bad Request: invalid parameters
- 401 Unauthorized: missing or invalid API key
- 429 Too Many Requests: rate limit exceeded
- 500 Internal Server Error: contact support

## Security Practices

### Overview
Best practices for keeping your account and data secure.

### Strong Password Guidelines
- Minimum 12 characters
- Mix of uppercase, lowercase, numbers, and symbols
- Avoid reusing passwords across sites
- Use a password manager (e.g., 1Password, Bitwarden)

### Recognizing Phishing
- We will NEVER ask for your password via email or chat
- Check sender email addresses carefully — official emails come from @example.com
- Do not click links in unsolicited emails — navigate directly to the site

### Security Incident Response
If you suspect your account is compromised:
1. Change your password immediately
2. Enable 2FA if not already active
3. Review Account > Security > Recent Login Activity
4. Contact support if you see unrecognized logins

## Integration Setup

### Overview
Connecting our platform to third-party tools like Slack, Zapier, and CRMs.

### Available Integrations
- **Slack**: Send ticket notifications to a Slack channel
- **Zapier**: Automate workflows between 5,000+ apps
- **Salesforce**: Sync customer data bidirectionally
- **HubSpot**: Link tickets to CRM contacts and deals

### Setting Up an Integration
1. Go to Account > Integrations
2. Find the integration and click "Connect"
3. Follow the OAuth flow or paste your API credentials
4. Configure trigger events and data mapping
5. Test with a sample event

### Troubleshooting Integrations
- **Not receiving events**: Check that webhook URLs are whitelisted in your firewall
- **Authentication errors**: Re-authorize the integration — tokens expire after 90 days
- **Data mismatch**: Review field mapping in the integration settings
